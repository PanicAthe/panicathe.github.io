# /commit

변경사항을 커밋한다.

## 실행 순서
1. `git status`와 `git diff`로 변경사항 파악
2. 변경 내용에 맞는 이모지 prefix로 커밋 메시지 초안 작성
3. 사용자에게 커밋 메시지 확인 후 승인받기
4. 관련 파일만 선택적으로 `git add`
5. `git commit` 실행
6. 필요하면 main 브랜치 push 여부 확인

## 이 프로젝트에서 특히 볼 것
- `dist/`가 변경되었으면 직접 수정인지 생성물인지 먼저 확인
- 문서 수정은 `📝`, UI 조정은 `💄`, 구조 개선은 `♻️`를 우선 고려
- 여러 성격의 변경이 섞였으면 한 커밋으로 몰지 말고 분리 가능성 검토
- 현재 환경에서 `git`이 safe.directory 문제로 막히면 사용자 로컬에서 실행 안내

## 커밋 메시지 형식
```text
<이모지> <변경 내용 요약>
```

| 타입 | 이모지 |
|------|--------|
| 새 기능 | ✨ |
| 버그 수정 | 🐛 |
| 스타일/UI | 💄 |
| 리팩토링 | ♻️ |
| 콘텐츠 수정 | 📝 |
| 이미지/에셋 | 🖼️ |
| 삭제 | 🔥 |
| 청소/잡일 | 🧹 |

## 주의
- `.env`, 비밀키 등 민감한 파일은 절대 포함 금지
- `git add .` 대신 파일을 명시적으로 지정
- 커밋 메시지는 반드시 사용자 승인 후 실행
